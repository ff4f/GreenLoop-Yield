5) 🏁 Claims Helper (Immersive/Impact)

Closed Journey 8 langkah (wajib lengkap), semua langkah ada output.

Pra-syarat otomasi:

Lot SETTLED; FT “ton” untuk lot tsb BURN atau transfer ke retirement acct (anti double-claim).

Langkah 1 — Select Lots & Year

UI: checklist lots eligible + input Claim Year (default tahun berjalan)

Output: ringkasan Total Tons (mis. 840 t)

Langkah 2 — Buyer Details

Fields: Buyer Legal Name, Jurisdiction Note (ops), CSR Link (ops)

Output: Claim JSON (draft) (preview on-screen)

Langkah 3 — Validate

Validasi status lot (SETTLED) + retire/burn FT “ton”

Output: label “Validated” + daftar Tx verifikasi (Mirror)

Langkah 4 — Generate Claim PDF

Render PDF (cover, summary, tabel lots, daftar tx hash, QR)

Output: HFS fileId_pdf 0.0.700567 (tampilkan) + tombol Download

Langkah 5 — Export Claim JSON

Simpan JSON final (isi sama dengan PDF)

Output: HFS fileId_json 0.0.700568

Langkah 6 — Anchor to Ledger (HCS)

Kirim SHA-256 dari PDF+JSON ke topic 0.0.900456

Output: AnchorTx Hash + Message Seq (mis. #1287)

Langkah 7 — Mint Claim Badge (Opsional)

Mint HTS NFT metadata (link fileId_pdf/json, topic_seq, total_tons, year)

Output: TokenId 0.0.600222 #1 → auto transfer ke wallet buyer

Langkah 8 — Share & Verify

Copy Proof Pack (JSON kecil) + Download ZIP (PDF+JSON)

Verify button: re-hitung hash lokal, cocokin dengan HCS; cek fileId hidup; cek NFT kepemilikan
